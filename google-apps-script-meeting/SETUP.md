# 한지붕 50+ 인연마당 신청 저장 연결

이 폴더의 Apps Script는 같은 Google 스프레드시트 안의 별도 탭 `50플러스_인연마당_신청`에 신청을 저장합니다. 기존 성경 말씀 및 사주풀이 신청 탭은 변경하지 않습니다.

대상 스프레드시트 ID: `1sxDS8D_SzMUfIcFFbJ-KMQciwu1CBW7oiAhoMd-_zJs`

## 1. Apps Script 프로젝트 준비

1. 대상 스프레드시트에서 `확장 프로그램 → Apps Script`를 엽니다. 별도 프로젝트로 분리해도 됩니다.
2. 기본 `Code.gs` 내용을 지우고 이 폴더의 `Code.gs` 전체를 붙여넣습니다.
3. 저장합니다.

## 2. 스크립트 속성 설정

Apps Script의 `프로젝트 설정 → 스크립트 속성`에 아래 값을 등록합니다.

| 속성 | 값 |
| --- | --- |
| `SPREADSHEET_ID` | `1sxDS8D_SzMUfIcFFbJ-KMQciwu1CBW7oiAhoMd-_zJs` |
| `SHEET_NAME` | `50플러스_인연마당_신청` |
| `CONSENT_VERSION` | `2026-07-12-meeting-v1` |
| `PARENT_ORIGINS` | `https://hanjibung.kr,https://www.hanjibung.kr,https://onehousetoge-hue.github.io` |

위 세 공개 사이트 주소는 코드에도 기본 허용값으로 들어 있습니다. 추가 개발 주소가 필요한 경우에만 `PARENT_ORIGINS`에 쉼표로 덧붙입니다. 운영 환경에서는 `*`를 사용하지 마세요.

## 3. 신청 탭과 헤더 준비

Apps Script 편집기 상단의 함수 목록에서 `setup`을 선택해 한 번 실행합니다. 최초 실행 시 Google 권한을 승인하면 `50플러스_인연마당_신청` 탭과 헤더가 자동으로 생성됩니다.

기존 탭에 다른 헤더가 들어 있다면 안전을 위해 실행이 중단됩니다. 그 경우 기존 데이터를 덮어쓰지 말고 탭 이름 또는 헤더 상태를 먼저 확인하세요.

## 4. 웹 앱 배포

1. `배포 → 새 배포 → 웹 앱`을 선택합니다.
2. 실행 사용자는 `나`, 액세스 사용자는 로그인하지 않은 신청자도 접근할 수 있는 범위로 설정합니다.
3. 배포 후 생성된 `/exec`으로 끝나는 웹 앱 주소를 복사합니다.
4. 웹사이트의 `meeting-config.js`에서 `appsScriptEndpoint`에 주소를 입력합니다.

```js
window.HANJIBUNG_MEETING_CONFIG = Object.freeze({
  appsScriptEndpoint: "https://script.google.com/macros/s/배포ID/exec",
  submitTimeoutMs: 20000,
  consentVersion: "2026-07-12-meeting-v1",
});
```

`/dev` 주소가 아니라 반드시 `/exec` 주소를 사용합니다. 코드를 수정했다면 새 버전으로 다시 배포해야 공개 웹 앱에 변경 사항이 반영됩니다.

## 5. 최소 접근 원칙

- 스프레드시트를 `링크가 있는 모든 사용자`에게 공개하지 않습니다. 웹 앱은 소유자 권한으로 실행하고 방문자는 시트 자체에 접근하지 않도록 합니다.
- 신청 탭의 편집 권한은 실제 접수 담당자에게만 부여하고, 가능하면 보호된 범위와 Google 계정 2단계 인증을 사용합니다.
- Apps Script의 스프레드시트 권한은 넓게 표시될 수 있습니다. 더 강하게 분리하려면 이 스프레드시트에만 편집 권한이 있는 전용 Google 계정으로 스크립트를 배포합니다.
- 코드에는 지정된 스프레드시트 ID만 사용하며, 불필요한 외부 서비스·라이브러리·OAuth 범위를 추가하지 않습니다.
- 이름, 나이, 거주 지역 등 신청 원문을 실행 로그에 남기지 않습니다.

## 6. 연결 확인

- 이름, 만 50~120세 나이, 시·도와 시·군·구를 정상 입력하면 한 행이 추가되는지 확인합니다.
- `만 50세 이상`, `현재 솔로`, `이용규칙·안전수칙`, `개인정보 수집·이용` 네 항목 중 하나라도 동의하지 않으면 저장되지 않는지 확인합니다.
- 같은 신청 버튼을 빠르게 두 번 눌러도 같은 `request_id`가 한 행만 저장되는지 확인합니다.
- 새 행의 `처리상태`가 `신청접수`이고 `카카오확인일시`, `입장안내일시`, `파기예정일`은 비어 있는지 확인합니다.
- `meeting-config.js`와 Apps Script 속성의 `consentVersion` 값이 동일한지 확인합니다.
- 봇 방지용 `website` 필드가 채워진 요청은 성공처럼 응답하지만 개인정보가 저장되지 않는지 확인합니다.

정식 운영 전 개인정보 보유기간과 파기 시점을 확정하고, 실제 정책에 맞춰 개인정보 동의문과 `파기예정일` 운영 절차를 마련하세요.
